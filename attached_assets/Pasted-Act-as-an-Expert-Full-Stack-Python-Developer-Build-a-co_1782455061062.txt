Act as an Expert Full-Stack Python Developer. Build a complete automated digital product store with a Telegram Bot integration.

**Tech Stack:**
- Backend: Python (FastAPI)
- Frontend: HTML, JS, Tailwind CSS
- Database: SQLite (Use SQLAlchemy ORM)
- Bot: pyTelegramBotAPI or aiogram (Webhook preferred)

**Project Structure Needed:**
1. `main.py` (FastAPI routes & Bot logic)
2. `database.py` (SQLAlchemy setup)
3. `templates/index.html` (Customer Storefront)
4. `templates/admin.html` (Admin Web Panel)

**Core Features & Flow:**

**1. Customer Storefront (index.html):**
- UI: Clean, modern E-commerce style. Display products as beautiful cards.
- Auth: Users MUST log in using the official Telegram Login Widget. No standard email/password.
- Product Fetching: Dynamically load products from SQLite.
- FOMO Effect: If a product has a 'fake_discount_price', show it crossed out next to the real price. Use JS `localStorage` to create a fake 15-minute countdown timer per user.
- Checkout: When clicking "Buy", open a modal to upload a payment slip (image) or paste a TrueMoney link.
- Submission: Send payload (User Telegram ID, Product ID, Payment Proof) to the Backend API. Show success message: "Payment submitted. Please wait for the bot's DM."

**2. Telegram Bot Admin Verification:**
- The bot sends the payment proof, Customer's Telegram ID, and requested Product Name to a specific Admin Group Chat.
- Include Inline Keyboard Buttons: `[ ✅ Approve ]` and `[ ❌ Reject ]`.
- If `Approve`: The bot reads the Product's associated `telegram_group_ids` from SQLite. It generates a single-use invite link (`createChatInviteLink(member_limit=1)`) for EACH group.
- The bot sends these invite links directly to the Customer's Telegram DM.

**3. Admin Web Panel (admin.html):**
- Auth: Protected route. To log in, the admin enters their Telegram ID on the web -> Bot sends an OTP to the Admin Group Chat -> Admin enters OTP on the web to access the dashboard.
- Features: A CRUD interface (no coding required for future updates).
- Admin can: Add, Edit, Delete products.
- Product Model Fields: `id`, `name`, `description`, `price`, `fake_discount_price` (nullable), `image_url` (nullable), `telegram_group_ids` (comma-separated string for bundle support, e.g., "group_id_1,group_id_2").

Write the complete code for this system, ensuring error handling and modularity. Start with the database models and FastAPI setup.
